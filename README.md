# My-Sixth-Task
This task focuses on performing a basic security analysis of Android applications to identify common vulnerabilities and insecure coding practices. The analysis is conducted in a safe and controlled environment using intentionally vulnerable applications such as InsecureBankv2 and DIVA.

# Basic Security Analysis of Vulnerable Android Applications

#Objective
Learn how Android application security testing works in a controlled lab environment.
Identify common vulnerabilities and insecure coding practices.
Understand how attackers exploit weak implementations.


# Practice secure coding
InsecureBankv2
Deliberately vulnerable banking app for Android security training.
Simulates real-world banking features.
Contains multiple security flaws for testing.
Damn Insecure and Vulnerable App (DIVA)
Educational Android app created for learning mobile app penetration testing.
Includes vulnerable modules covering storage, IPC, SQL injection, and more.
Widely used in Android security labs.

# Safe Testing Environment Setup

# Educational Importance
Helps students understand mobile application threats.
Demonstrates real-world attack techniques safely.
Improves secure coding awareness.
Supports penetration testing learning.

# Ethical Considerations
Perform testing only in authorized environments.
Never attack real applications without permission.
Use vulnerable apps strictly for education and research.
Follow responsible disclosure practices.
